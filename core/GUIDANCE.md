# Core Guidance for AI-Developer Collaboration

This document outlines universal principles for effective collaboration between developers and AI assistants.

## Communication Principles

1. **Be Clear and Specific** - State exactly what you need
2. **Provide Context** - Include relevant framework, language, and constraints
3. **Chunk Work** - Break tasks into manageable pieces that respect context windows
4. **Use Consistent Terminology** - Align with framework/language conventions

## Workflow Patterns

1. **Exploration → Planning → Implementation** - Follow this sequence for complex tasks
2. **Iterative Refinement** - Start simple, then enhance in subsequent iterations
3. **Position-Based Reasoning** - Reference code by line numbers or structural elements
4. **Template-Based Communication** - Use consistent patterns for requesting changes

## Context Management

1. Load only relevant framework/language guidelines when needed
2. Explicitly reference which standards apply to the current work
3. Prioritize documentation over code examples when explaining concepts
4. Use cursor-based navigation when discussing specific code
